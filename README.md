## 🔍 О проекте

Данное приложение реализует классическую задачу теории графов: по данному дереву построить его **код Прюфера** (последовательность длины `n-2`) и, обратно, по коду Прюфера восстановить исходное дерево.

**Ключевая особенность:** алгоритмы работают в пошаговом режиме с визуальной анимацией на графе и детализированным текстовым логом на русском языке, что позволяет наглядно изучать процесс кодирования и декодирования.

### Что такое код Прюфера?
Код Прюфера — это способ однозначного представления помеченного дерева с `n` вершинами в виде последовательности длины `n-2`. Между деревьями и последовательностями существует биекция, что делает код Прюфера важным инструментом в комбинаторике и алгоритмах на графах.

### Области применения
- 🎓 Образовательные цели: изучение кода Прюфера и свойств деревьев
- 🔬 Исследования: комбинаторная генерации всех деревьев на `n` вершинах
- 🛠️ Инженерия: визуальная отладка алгоритмов на деревьях

## ✨ Возможности

### 🎨 Frontend (Интерфейс)
| Функция | Описание |
|---------|----------|
| **Интерактивный редактор** | Добавление вершин кликом по холсту, создание рёбер через выделение пары узлов |
| **Удаление элементов** | Правый клик по вершине (с каскадным удалением инцидентных рёбер) или по ребру |
| **Автоматическая нумерация** | Вершины получают метки V1, V2, ..., Vn в порядке создания |
| **Пошаговая анимация** | Таймер (700ms) автоматически проходит шаги алгоритма с подсветкой активных элементов |
| **Подсветка активных элементов** | На каждом шаге подсвечиваются узлы и рёбра, участвующие в текущей итерации |
| **Панель управления** | Кнопки `Кодирование` / `Декодирование` / `Стоп` и окно лога |
| **Лог алгоритма** | Текстовое описание каждого шага на русском языке |

### ⚙️ Backend (Алгоритм)
| Компонент | Описание |
|-----------|----------|
| **Кодирование (encode)** | Итеративное удаление минимального листа с записью его соседа в последовательность |
| **Декодирование (decode)** | Восстановление дерева по последовательности через степени вершин |
| **Валидация дерева** | Проверка связности, количества рёбер, минимального числа вершин |
| **Автоматическая раскладка** | При декодировании вершины располагаются DFS-алгоритмом для наглядного отображения |
| **Детерминированный трейс** | Полная история изменений состояния для пошаговой визуализации |
| **Независимость от GUI** | Модуль алгоритма может использоваться отдельно от интерфейса |

### Поток данных
1. Пользователь строит дерево в интерактивном редакторе (клики по холсту)
2. При нажатии **Кодирование** — `PruferAnimator` считывает вершины и рёбра из `GraphScene`, вызывает `PruferAlgorithm.encode()`, получает массив шагов и запускает `QTimer`
3. По таймеру вызывается `_next_step()` — подсвечивает/удаляет элементы на сцене, пишет сообщение в лог
4. При нажатии **Декодирование** — появляется диалог ввода последовательности, вызывается `PruferAlgorithm.decode()`, сцена очищается, строится новое дерево, запускается анимация добавления рёбер
5. Кнопка **Стоп** останавливает таймер в любой момент

## 📐 Формат данных

### Входные данные для decode
Последовательность кода Прюфера вводится в диалоговом окне как строка с числами, разделёнными запятыми:
```
1, 1, 2
```

### Структура шага трейса
Каждый шаг визуализации — это словарь со следующими полями:

```python
{
    "type": "highlight_v" | "highlight_e" | "remove_v" | "remove_e" | "add_e" | "reset_colors" | "log" | "finish",
    "target": str | None,      # Метка вершины или ключ ребра
    "message": str | None,     # Текстовое описание шага
    "result": str | None,      # Финальный код Прюфера (только для type="finish")
    "v1": str | None,          # Первая вершина ребра (для add_e)
    "v2": str | None           # Вторая вершина ребра (для add_e)
}
```

### Валидация дерева (encode)
| Условие | Проверка |
|---------|----------|
| Минимум 2 вершины | `len(vertices) >= 2` |
| Количество рёбер | `len(edges) == n - 1` |
| Связность | BFS от первой вершины достигает всех остальных |

## 💡 Примеры использования

### Пример 1: Кодирование дерева

Постройте дерево из 4 вершин:
```
V1 — V2 — V3 — V4
```

Нажмите **Кодирование**. Ожидаемый результат в логе:
```
Текущее дерево: V1, V2, V3, V4
Рёбра: V1-V2, V2-V3, V3-V4
Начинаем кодирование.
Наименьший лист: V1, его сосед: V2
Удаляем V1 и ребро V1-V2. Код: [V2]
Наименьший лист: V3, его сосед: V2
Удаляем V3 и ребро V3-V2. Код: [V2, V2]
Кодирование завершено. Код Прюфера: [V2, V2]
```

### Пример 2: Декодирование последовательности

Нажмите **Декодирование**, введите:
```
2, 2, 3
```

Приложение построит дерево из 5 вершин с рёбрами:
- Сначала соединит V1 с V3, затем V2 с V3, затем V4 с V2
- Финальное дерево: V1—V3—V2—V4 и V3—V5

### Пример 3: Ошибка "граф не является деревом"

Постройте граф с циклом (3 вершины, 3 ребра) или несвязный граф и нажмите **Кодирование**.

Ожидаемый результат: сообщение в логе об ошибке с указанием причины (неверное количество рёбер или граф не связен).

## ⚠️ Обработка ошибок

### На стороне клиента (GUI)

При возникновении ошибки приложение:
1. Выводит красное сообщение в лог-панели
2. Блокирует повторный запуск анимации до её остановки

### Сценарии ошибок

| Сценарий | Обработка |
|----------|-----------|
| Меньше 2 вершин | Сообщение: "Граф должен содержать хотя бы 2 вершины" |
| Неверное число рёбер | Сообщение: "Граф должен содержать ровно n-1 рёбер для дерева" |
| Несвязный граф | Сообщение: "Граф не является связным" |
| Повторный запуск анимации | Игнорируется (флаг `running` блокирует повторный вход) |
| Пустая последовательность | Сообщение: "Последовательность не может быть пустой" |

## 🔧 Расширение функционала

### Регулировка скорости анимации
```python
# В frontend/prufer_animator.py — изменить интервал таймера
self.timer = QTimer()
self.timer.setInterval(500)  # 500ms вместо 700ms
```

### Экспорт лога в файл
```python
# В PruferAnimator добавить метод:
def export_log(self, filename):
    with open(filename, "w", encoding="utf-8") as f:
        f.write(self.log_widget.toPlainText())
```

### Импорт графа из JSON
```python
# В GraphScene добавить:
def load_from_json(self, data):
    self.clear_all()
    for node in data["nodes"]:
        self.add_vertex_at(node["x"], node["y"], node["label"])
    for edge in data["edges"]:
        self.add_edge_by_labels(edge["v1"], edge["v2"])
```

### Сохранение состояния графа
При закрытии окна можно автоматически сохранять граф в JSON и восстанавливать при следующем запуске через переопределение `closeEvent` в `MainWindow`.
